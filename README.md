Проект "Анализ эффективности новой платежной системы и оптимизация конверсии"

Проект представляет собой полный цикл аналитического исследования с использованием A/B-тестирования для оценки влияния новой механики оплаты на ключевые метрики монетизации. Работа включала критический анализ качества данных, выбор робастных статистических методов (Бутстрап) и формирование обоснованного бизнес-решения.

Цели проекта:

Оценить эффективность: Определить, влияет ли новая механика оплаты на конверсию (CR), средний чек платящих пользователей (ARPPU) и общий доход на пользователя (ARPU).

Проверить значимость: Установить, являются ли наблюдаемые различия между контрольной (A) и тестовой (B) группами статистически значимыми.

Принять бизнес-решение: Сформулировать окончательную рекомендацию о масштабировании механики на всех пользователей.

Обеспечить надёжность: Выявить и устранить аномалии в исходных данных, влияющие на чистоту эксперимента.

Направление,Библиотеки,Навыки:

Обработка данных,"Pandas, NumPy, operator","ETL-процессы, очистка и агрегация данных, обработка пропусков."

Робастный анализ,NumPy (для генерации выборок),"Реализация метода Бутстрапа для сравнения средних значений (ARPU, ARPPU) в условиях ненормального распределения."

Визуализация,"Matplotlib, Seaborn","Построение сравнительных барплотов, Q-Q Plot, гистограмм; создание отчётной визуализации с P-value."

Автоматизация,Python Functions,"Создание функций для автоматической загрузки, очистки и пересчёта метрик."

Ключевые задачи и логика решения:

Сборка и очистка данных: Объединила четыре CSV-файла, включая обработку дополнительного файла с группами. Применила унификацию колонок для гибкой загрузки.

Устранение Data Quality Аномалии: Выявила 149 пользователей, оплативших, но неактивных. Для корректности анализа временно классифицировала их как активных, но дала критическую рекомендацию по расследованию и устранению системного сбоя в логировании.

Выбор статистического теста: Проведена проверка предпосылок (ненормальность распределения доходов и гетероскедастичность). Выбраны робастные непараметрические тесты (Z-тест для CR, Бутстрап для ARPU/ARPPU) как наиболее подходящие для сильно скошенных финансовых метрик.

Расчет и сравнение метрик: Сравнение по Конверсии (CR), Среднему чеку (ARPPU) и Общему доходу на пользователя (ARPU).

Финальная рекомендация:

Бизнес-решение: Не рекомендуется масштабировать новую механику в её текущем виде. Хотя она успешно увеличивает средний чек платящих клиентов (ARPPU), этот эффект недостаточен, чтобы обеспечить статистически значимый рост общего дохода (ARPU). Требуется доработка механики с фокусом на улучшение конверсии.

Data Quality Action: Дано требование немедленно исправить системную ошибку логирования, приведшую к аномалии "платящих, но неактивных" пользователей, так как ненадёжные данные ставят под угрозу точность любых экспериментов.
